# 배포 가이드 (3단계)

> 예상 소요 시간: 약 20~30분  
> 비용: 전부 무료 (Upstash 무료 플랜, Vercel 무료 플랜, Google OAuth 무료)

---

## STEP 1 — Upstash Redis 설정 (임시 데이터 저장소)

1. https://upstash.com 접속 → **Sign Up** (GitHub 계정으로 가입 가능)
2. **Create Database** 클릭
   - Name: `figma-auth` (아무거나)
   - Region: `ap-northeast-1` (도쿄, 한국에서 가장 빠름)
   - Type: **Regional**
3. 생성 후 **REST API** 탭 클릭
4. 아래 두 값을 복사해 어딘가에 메모
   - `UPSTASH_REDIS_REST_URL` (예: `https://xxx.upstash.io`)
   - `UPSTASH_REDIS_REST_TOKEN` (예: `AXxx...`)

---

## STEP 2 — Google OAuth Client ID 발급

1. https://console.cloud.google.com 접속 → 새 프로젝트 생성
   - 프로젝트 이름: `figma-spec-auth` (아무거나)

2. 왼쪽 메뉴 → **APIs & Services** → **OAuth consent screen**
   - User Type: **External** 선택
   - App name: `화면 정의서 생성기`
   - User support email: 본인 이메일
   - Developer contact information: 본인 이메일
   - **저장 후 계속** (나머지 스텝도 기본값으로 계속)

3. 왼쪽 메뉴 → **Credentials** → **CREATE CREDENTIALS** → **OAuth 2.0 Client ID**
   - Application type: **Web application**
   - Name: `figma-plugin` (아무거나)
   - **Authorized JavaScript origins** 에 추가:
     - `https://figma-spec-auth.vercel.app` ← 나중에 실제 도메인으로 교체
     - (Vercel 배포 후 실제 URL을 여기에 추가해야 함)
   - **CREATE** 클릭

4. 생성된 **Client ID** 복사 (예: `123456789-abc.apps.googleusercontent.com`)

5. `web/auth.html` 파일에서 아래 줄을 찾아 실제 값으로 교체:
   ```javascript
   var GOOGLE_CLIENT_ID = 'YOUR_GOOGLE_CLIENT_ID.apps.googleusercontent.com';
   // ↓ 교체
   var GOOGLE_CLIENT_ID = '123456789-abc.apps.googleusercontent.com';
   ```

---

## STEP 3 — GitHub + Vercel 배포

### 3-1. GitHub 저장소 생성

1. https://github.com/new 에서 새 저장소 생성
   - Repository name: `figma-spec-auth`
   - Visibility: **Private** 권장
   - **Create repository** 클릭

2. 터미널에서 `web/` 폴더를 GitHub에 푸시:
   ```bash
   cd /Users/ihaebin/Desktop/FIGMA/web
   git init
   git add .
   git commit -m "init"
   git remote add origin https://github.com/YOUR_USERNAME/figma-spec-auth.git
   git push -u origin main
   ```

### 3-2. Vercel 배포

1. https://vercel.com 접속 → GitHub 계정으로 로그인
2. **Add New Project** → 방금 만든 `figma-spec-auth` 저장소 선택
3. **Framework Preset**: Other (자동 감지됨)
4. **Environment Variables** 섹션에 추가:

   | Key | Value |
   |-----|-------|
   | `UPSTASH_REDIS_REST_URL` | Step 1에서 복사한 URL |
   | `UPSTASH_REDIS_REST_TOKEN` | Step 1에서 복사한 토큰 |

5. **Deploy** 클릭
6. 배포 완료 후 도메인 확인 (예: `https://figma-spec-auth.vercel.app`)

### 3-3. Google Console에 실제 도메인 등록

1. Google Cloud Console → Credentials → 방금 만든 Client ID 수정
2. **Authorized JavaScript origins** 에 실제 Vercel 도메인 추가:
   - `https://figma-spec-auth.vercel.app` (실제 도메인으로)

---

## STEP 4 — 플러그인 코드 업데이트

`code.js` 파일 상단의 URL 변수를 업데이트:

```javascript
// 변경 전
var AUTH_PAGE_URL = '';
var API_BASE_URL  = '';

// 변경 후 (실제 도메인으로 교체)
var AUTH_PAGE_URL = 'https://figma-spec-auth.vercel.app/auth.html';
var API_BASE_URL  = 'https://figma-spec-auth.vercel.app';
```

---

## 완료 확인

1. 브라우저에서 `https://YOUR_DOMAIN.vercel.app/auth.html?state=test123` 접속
2. Google로 계속하기 버튼 클릭 → 로그인 완료 메시지 확인
3. 브라우저에서 `https://YOUR_DOMAIN.vercel.app/api/auth-result?state=test123` 접속
   - `{ "found": true, "user": { "name": "...", "email": "..." } }` 응답 확인

---

## 문제 해결

**"Google 로그인 버튼이 안 뜸"**  
→ Google Client ID가 잘못됐거나, Vercel 도메인이 Authorized JavaScript origins에 없음

**"로그인 후 피그마에 반영 안 됨"**  
→ `API_BASE_URL`이 code.js에 설정됐는지 확인, Upstash 환경변수 확인

**"환경 변수 오류"**  
→ Vercel 대시보드 → Settings → Environment Variables 확인
